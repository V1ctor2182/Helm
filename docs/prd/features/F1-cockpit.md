# F1 · Coding Agent 驾驶舱

> 来源：FanBox｜优先级：**P0**（核心子集）/ P1（进阶）｜里程碑：M1 / M5
> 上级：[PRD 主文档](../00-PRD-Master.md)

把 FanBox 的「**文件 × 终端 × 原地预览**」驾驶舱作为 Helm 的左/右两栏与核心交互。这是 Helm 区别于普通 AI 工作台的灵魂：**看得见 agent 干了什么，随时接管。**

---

## 1. 目标
- 在一个窗口里完成「**找回文件 → 预览 → 轻改 → 指挥 agent → 看清改动**」。
- 实时观测任意 coding agent（Claude Code / Codex / 任意写文件的进程）对文件系统的改动。
- 不跟 Finder 拼文件管理、不跟 VS Code 拼重度编辑。

## 2. 非目标
- 不做完整 IDE、不做项目构建系统、不做调试器。

---

## 3. 功能拆解

### 3.1 文件 · 找回与预览（P0）
- **⌘K 全局模糊搜索**：记得名字片段就能找到；`⌘↵` 用外部编辑器整包打开项目；`内容:关键词` 切全文搜索。
- **强色实体图标**：每种文件「长得像它自己」（PDF 红、JS 黄、Markdown 蓝），照片/视频按真实比例。
- **原地预览**：Markdown 渲染、HTML 实时成品、代码语法高亮、图片/视频/PDF 内嵌（含 HEIC）、压缩包内容清单、透明图棋盘格。
- **缩略图加速**：大文件夹滚动/点击 < 0.1s。
- **项目徽章**：文件夹卡片标 node / web / py / rs / go，一眼认出项目类型。

### 3.2 看 agent 改了什么（P0 核心：高亮+diff；P1：回放+收件箱）
- **活的仪表盘（P0）**：agent 每写一个文件，对应卡片涟漪/呼吸发光，光跟着 agent 走。
- **跟随模式（P0）**：文件视图 + 预览自动跟踪 agent 正在编辑的文件；代码随新写行高亮，HTML 双缓冲实时渲染（零白闪），Markdown 实时渲染；任何手动浏览立即交还控制权。
- **Git diff（P0）**：Monaco 只读 DiffEditor，HEAD vs 工作区并排。
- **会话回放（P1）**：拖时间轴像刷视频一样重现 agent 一步步改了哪些文件。
- **变更收件箱（P1）**：跨多个项目汇总本会话所有被改动的文件。

### 3.3 终端 · 指挥 agent（P0）—— 与 F5 协同
- 内嵌**真实终端**（后端 pty + WebSocket → xterm.js），在选定项目目录里跑 `claude` / `codex` / 任意命令。
- 终端与文件监听联动：终端里 agent 写文件 → 仪表盘/跟随模式即时反应。
- 详细的 agent 编排（外部 CLI vs 内置 agent、MCP 注入）见 [F5](./F5-agent-orchestration.md)。

### 3.4 项目记忆与续接（P1）
- 打开任一项目文件夹，看 AI 在这里干过什么：历史会话（你的第一句话当标题）、每次会话改过的文件、触发过的 skill。
- **「▶ 续上」**：一键在内嵌终端 `claude --resume` / `codex resume` 接回当时上下文。

### 3.5 辅助（P1/P2）
- **截图直通车**：系统截屏落盘即浮卡 → 喂给终端 agent / 收进项目 `素材/` / 标注后发。
- **AI 整理**（P2）：AI 只看元数据出整理提案（不读内容、不碰 FS），逐条过人、执行带回滚日志、一键撤销。
- **磁盘占用透视 / 发版向导 / Skills 用量**（P2 backlog）。

---

## 4. 交互与布局
- **左栏**：文件浏览（卡片/列表）+ ⌘K 搜索入口 + 项目切换。
- **中央**：原地预览 / Monaco diff / 轻编辑（与大脑侧的 Chat/Research 共用中央区，标签切换）。
- **右栏（或下栏）**：终端（可多标签会话）。
- **跟随模式开关**显著可见；进入/退出有明确视觉反馈。

## 5. 数据模型（关联 F0）
- `projects(path, type_badges, last_opened, ...)`
- `terminal_sessions(id, project_path, agent, started_at, ...)`
- `file_changes(id, session_id, path, change_kind, ts)` —— 驱动仪表盘/回放/收件箱
- 监听来源：watchdog 文件事件 + （可选）解析 agent 输出/会话日志。

## 6. 用户故事
- 作为用户，我能用 ⌘K 凭名字片段秒找上周 AI 起的项目。
- 作为用户，我能开跟随模式，盯着 agent 把 HTML 一行行写出来并实时看到渲染。
- 作为用户，agent 跑完后我能一眼 git diff 看清它改了哪几行。
- 作为用户，我能从某个项目「续上」上次的 Claude Code 会话。

## 7. 验收标准
- [ ] 大文件夹（>1000 项）滚动与点击响应 < 0.1s。
- [ ] 终端里 agent 写文件后，对应卡片高亮延迟 < 500ms。
- [ ] 跟随模式下手动点击任意文件能立即夺回控制权。
- [ ] git diff 与 `git diff` CLI 结果一致。
- [ ] 预览支持：md / html / 常见代码 / png·jpg·heic / mp4 / pdf / zip 清单。

## 8. 依赖与风险
- **前端复用**：FanBox 驾驶舱前端（文件卡片/预览/跟随/回放/Monaco diff）直接搬入——这是最值钱且保留的部分。
- **后端重写（机械活）**：终端改后端 pty（`pexpect`）+ WebSocket；文件监听改 `watchdog`；前端通信从 Electron IPC 改为 REST/WebSocket（见 F0）。
- 风险：①「实时渲染零白闪」「跟随模式」是 FanBox 精细体验点，迁移通信层时需保真（双缓冲逻辑保留在前端）；②终端延迟需控制（pty + WS）。
