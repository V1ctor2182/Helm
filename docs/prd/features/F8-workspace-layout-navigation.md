# F8 · 工作台布局与导航（统一入口 / IA）

> 来源：新建（融合 UX）｜优先级：**P0**（贯穿 MVP 的统一外壳）｜里程碑：M0–M1
> 上级：[PRD 主文档](../00-PRD-Master.md)

回答一个核心问题：**Helm 是「我的 AI workspace」，怎么设计才能让我最方便地够到所有功能（驾驶舱、Chat、研究、记忆、日记/任务、邮件/日历）？**

本文件定义统一外壳的信息架构（IA）、布局、导航与全局入口。所有 feature（F1–F7）都"挂"在这套外壳上。

---

## 1. 设计目标
- **一两步到达任何功能**：不翻菜单、不找窗口。
- **一个窗口**：所有能力同窗，按需展开/收起，不来回切应用。
- **键盘优先**：高频操作都有快捷键；命令面板是万能入口。
- **上下文感知**：工作台始终知道"当前项目"，Chat/研究/agent/记录都能一键挂到它。
- **零摩擦捕获**：任何时候记一条想法/任务，不打断手头的事。

## 2. 非目标
- 不做高度可定制的「拖拽式仪表盘」（首版给一套好用的默认布局即可）。

---

## 3. 三大「万能入口」（这是"最方便"的答案）

### 3.1 命令面板（⌘K）—— 通往一切
一个搜索框，输入即可：
- **跳转**：到任意模式（Chat / 研究 / 记忆 / 记录 / 邮件 / 日历 / 驾驶舱）。
- **找东西**：项目、文件（含全文 `内容:`）、会话、笔记、记忆、邮件。
- **执行动作**：新建 Chat、发起一次研究、记一条速记、新建任务、续接某项目的 agent 会话、打开设置……
- **唤起 agent**：直接把一句话作为指令交给内置 agent / 终端 agent。
> 一个 ⌘K 覆盖「导航 + 搜索 + 动作」，是"够到所有功能"的主路径。

### 3.2 全局速记快捷键 —— 通往捕获
任意界面一键弹出输入框，记完即走，自动归入速记（可一键转任务/记忆/日记，见 F6）。

### 3.3 左侧模式栏（Rail）—— 通往各大区
一条窄的图标栏，常驻最左，点一下切换中央工作区的"模式"。鼠标用户的主路径，与 ⌘K 互补。

## 4. 布局（默认）

```
┌────┬───────────────┬──────────────────────────┬───────────────┐
│ R  │  上下文面板    │       中央工作区          │   终端面板     │
│ a  │ （随模式变）   │   （Tab 化，可放任意功能）│  （可收起）    │
│ i  │                │                          │               │
│ l  │  驾驶舱模式→   │  Chat / 研究报告 / 文档   │  pty + xterm  │
│    │   文件浏览     │  / Diff / 邮件 / 日历     │  跑 claude/   │
│ 🏠 │  记录模式→     │  / 日记 / 任务            │  codex        │
│ 💬 │   笔记/日记列表 │                          │               │
│ 🔍 │  邮件模式→     │                          │               │
│ 🧠 │   邮件列表     │                          │               │
│ 📓 │                │                          │               │
│ ✉  │                │                          │               │
│ ⚙  │                │                          │               │
└────┴───────────────┴──────────────────────────┴───────────────┘
  ↑ 模式栏        ↑ 左面板与右终端均可收起 → 纯沉浸（如全屏 Chat / 研究）
```

- **左侧 Rail（模式栏）**：🏠 Today · 💬 Chat · 🔍 Deep Research · 🧠 记忆 · 📓 记录(日记/速记/任务) · ✉ 邮件/日历 · （驾驶舱图标）· ⚙ 设置。
- **上下文面板（第二列）**：内容随模式变（驾驶舱=文件浏览/预览；记录=笔记列表；邮件=邮件列表…）。可收起。
- **中央工作区**：**Tab 化**，一个 Tab 可以是 Chat、研究报告、文档、Diff、邮件正文、日历、日记——多功能并存、可并排。
- **终端面板（最右/下）**：随时可收起；驾驶舱场景常驻。
- **自适应**：做纯 Chat/研究时一键收起左右面板进入沉浸；做编程时展开文件+终端。

---

## 5. Today 主页（🏠）—— 一眼掌握 + 快速进入
打开 Helm 的落地页，聚合"现在该看/该做的"，每块都可点进去：
- **今日任务 / 到期提醒**（F6）
- **写两行日记** 的入口 + AI「今日小结」按钮（F6）
- **最近项目**（卡片，点开即进驾驶舱并可「续上」会话，F1）
- **agent 最近活动 / 变更收件箱**（F1）
- **紧急/未读邮件**（AI 已分诊，F7）
- **快速动作**：新 Chat、发起研究、记一条。

## 6. 上下文感知（贯穿所有功能）
- 工作台维护一个「**当前项目**」概念。
- 在任意模式都能：把当前项目/文件/选区作为 Chat 上下文、作为研究输入、关联到任务/日记。
- agent（F5）、记忆（F4）、记录（F6）都能读到"当前在干什么"。

## 7. 键盘地图（示例，可调）
- `⌘K` 命令面板（万能） · `⌘N` 速记 · `⌘⏎` 把输入交给 agent
- `⌘1..7` 切换模式（Today/Chat/研究/记忆/记录/邮件/驾驶舱）
- `⌘P` 项目切换 · `⌘\` 收起/展开侧栏 · `` ⌘` `` 聚焦终端

## 8. 用户故事
- 作为用户，我按 ⌘K 输入几个字就能跳到任何功能或执行任何动作。
- 作为用户，我打开 Helm 看到 Today 主页，今天要做什么、最近在搞哪些项目一目了然。
- 作为用户，我在写代码时展开文件+终端，转去查资料时一键收起进入沉浸式研究。
- 作为用户，我在任意界面按一个键就能记下闪过的想法。
- 作为用户，我发起的 Chat / 研究自动带上我当前项目的上下文。

## 9. 验收标准
- [ ] ⌘K 能跳转到全部模式、搜到项目/文件/会话/笔记/记忆/邮件，并执行核心动作。
- [ ] 任意功能从打开 App 起 ≤ 2 步可达（Rail 点击 或 一次 ⌘K）。
- [ ] 全局速记快捷键在任意界面 < 1s 唤出。
- [ ] 左右面板可收起，进入纯沉浸模式。
- [ ] Today 主页各模块可点击进入对应功能并带上下文。
- [ ] 切换模式不丢失中央工作区已打开的 Tab。

## 10. 依赖与风险
- 依赖 F0（外壳/前端框架）；统领 F1–F7 的呈现方式。
- 风险：功能多，导航若堆砌会变复杂 —— 用「⌘K 万能入口 + 精简 Rail + Today 聚合」三招控制复杂度，避免深层菜单。
- 与 F1 的三栏布局对齐：驾驶舱是「模式」之一，文件/终端面板是可收起的通用面板。
